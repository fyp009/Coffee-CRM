## Coffee CRM

Coffee CRM —  приложение на JavaFX для управления кофейной точкой. Оно объединяет учет заказов, клиентов и ассортимента, работу персонала (администратор, кассир/бариста) и историю операций в одном удобном интерфейсе. База — встроенная SQLite, поэтому система работает локально без отдельного серверного развертывания.

## Технологии

- Java 23, JavaFX 25 (FXML-интерфейсы, таблицы, сцены, стили)
- SQLite (через JDBC), схема создаётся автоматически
- Структура модулей: Admin (админ-панель), User (клиентский интерфейс)
- Макеты и контроллеры: FXML + контроллеры Java, таблицы и гриды для данных

## Роли и права

- Администратор:
  - авторизация в админ-панель;
  - подтверждение новых заказов;
  - управление «в ожидании»/«принятыми»/«завершёнными» заказами;
  - просмотр истории по датам;
  - управление ассортиментом (цены, изображения напитков);
  - создание пользователей (сотрудников);
  - смена пароля;
  - система статусов (Online/Offline) при входе/выходе.
- Кассир/Бариста:
  - принимает подтверждённые заказы («Принять»);
  - завершает выполнение («Готово»);
  - видит детализацию напитков и сумм.

- Клиент:
  - просмотр меню (сетка карточек напитков);
  - выбор типа напитка (Hot/Iced/Frappee) и количества;
  - добавление в корзину;
  - оформление заказа (подтверждение);
  - после заказа — оставление оценки (1–5 звёзд) и комментария;
  - регистрация (Sign Up) и вход (Login).

## Основные разделы

- Главный экран (DefaultInterface.fxml): вход в админку, оформление заказа, регистрация/вход клиента.
- Клиентский интерфейс (UserInterface.fxml):
  - список напитков (из `drink_list`);
  - выбор типа/количества, текущее изображение и цены;
  - «корзина» (временная таблица `temp_customer`);
  - подсчёт итоговой суммы; удаление и сброс корзины;
  - оформление заказа с подтверждающим диалогом.
- Админ-панель:
  - Accepted / Confirm Orders (новые «Ordered» — на подтверждение);
  - Pending Orders (подтверждённые «Pending» — на принятие/отмену/завершение);
  - History (история «Completed» с фильтрами по датам, детализация по клиенту);
  - Update Drink (ассортимент);
  - Create User (создание сотрудников);
  - Change Password;
  - Logout (возврат статусов и безопасный выход).

## Жизненный цикл заказа

1. Ordered — клиент оформил заказ (корзина переносится в историю).
2. Pending — администратор подтвердил заказ (готов к началу работы).
3. Accepted — кассир/бариста принял заказ к исполнению.
4. Completed — заказ выполнен и закрыт.
5. Canceled — заказ отменён на любой стадии.

История (History) показывает агрегированные итоги по клиентам и датам, детализация раскрывает состав заказов (тип, напиток, цена, количество, сумма).

## База данных (основные таблицы)

- `admin_account`: пользователи-администраторы/сотрудники (логин, роль, статус).
- `drink_list`: меню (цены для Hot/Iced/Frappee, путь к изображению).
- `customers`: клиенты (имя, email, пароль, дата регистрации, статус).
- `history`: журнал заказов (напиток, тип, цена, количество, итог, дата/время, статус, подтверждающий/принявший).
- `feedback`: оценки 1–5 и комментарии после заказа (привязка к клиенту).
- `temp_customer`: временная корзина клиента (до оформления).
- `temp_admin`, `temp_drinklist`: вспомогательные временные таблицы.

## Функционал

- Полный цикл заказов (от корзины до истории).
- Разделение ролей и действий (админ, кассир/бариста, клиент).
- Регистрация/вход клиентов, вход администратора.
- Система отзывов и рейтингов после заказа.
- История с фильтром по диапазону дат и детализацией.
- Локальная работа на SQLite (без внешнего сервера), улучшенная устойчивость к блокировкам (режим WAL).
